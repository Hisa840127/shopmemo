重要
【このクラスは何をしている？】

Spring Boot起動時に、

初期データ（卵・牛乳）を登録するクラス。

DBが空の場合のみデータを投入する。

【最低限どこを見る？】

@Configuration

↓

Springの設定クラスのタグ

@Bean

CommandLineRunner loadData(...)

↓

起動時に実行する処理をSpringへ登録

itemRepository.save(...)

↓

DBへデータ保存

【ざっくり流れ】

Spring起動

↓

DataLoader発見

↓

loadData()実行

↓

CommandLineRunner取得

↓

Springが run() 実行

↓

初期データ登録

【覚えておけば十分なこと】

・初期データ登録用クラス

・@Configuration は

「設定クラス」のタグ

・CommandLineRunner は

起動時に1回実行される

・itemRepository.save(...)

でDBへ保存している

・細かい補足を忘れても、

「起動時に卵と牛乳を登録するクラス」

と分かればOK

補足1　CommandLineRunner

CommandLineRunner は

Spring Boot 起動時に1回実行される処理。

return args -> {
    // 処理
};

はラムダ式。

今回は、

return new CommandLineRunner() {
    @Override
    public void run(String... args) {
        // 処理
    }
};

を省略して書いている。

args は

run(String... args)

の引数。

main(String[] args)

の args と似たもの。

今回は受け取っているだけで使用していない。

return しているのは args ではなく、

「CommandLineRunner の実装（ラムダ式全体）」。

今回の処理内容は、

DBにデータが1件も無い場合のみ

初期データ（卵・牛乳）を登録すること。

==================================================

補足2　loadData() の役割

loadData() は

CommandLineRunner を返すメソッド。

戻り値の型は CommandLineRunner。

@Bean が付いているため、

Spring は起動時に loadData() を呼び出す。

loadData() が返しているのは、

CommandLineRunner のインスタンス

（今回はラムダ式で作成）。

イメージ

Spring起動

↓

loadData() 実行

↓

CommandLineRunner取得

↓

Springが保持（管理）

↓

Springが run() 実行

このクラス内には

run() の呼び出しは書かれていない。

run() を呼び出しているのは Spring 側。

そのため、

CommandLineRunner loadData(...)

の

CommandLineRunner

は

loadData() の戻り値の型。

loadData() 自体は、

「起動時に実行する処理（CommandLineRunner）を作って返す工場」

のような役割。

==================================================

補足3　ItemRepository が使える理由

ItemRepository は

クラスではなくインターフェース。

public interface ItemRepository
        extends JpaRepository<Item, Long> {
}

のように定義されている。

通常のJavaでは、

インターフェースだけでは

インスタンスを作れない。

例

Printer printer = new Printer(); // コンパイルエラー

Spring Data JPA では、

ItemRepository

↓

JpaRepository を継承

していることを Spring が検知し、

save()

findAll()

findById()

deleteById()

count()

などを実装したクラスを

裏側で自動生成してくれる。

さらに、

Spring がそのインスタンスも作成し、

必要な場所へ渡してくれる（DI）。

そのため、

CommandLineRunner loadData(ItemRepository itemRepository)

のように書くだけで、

itemRepository.save(...)

を利用できる。

イメージ

ItemRepository

↓

Repositoryの設計図（インターフェース）

↓

Spring

↓

実装クラス自動生成

↓

インスタンス作成

↓

DI

↓

itemRepository.save(...)

覚えておけば十分なこと

・ItemRepository はインターフェース

・Spring が実装クラスを自動生成する

・Spring がインスタンスも作って渡してくれる

・そのため itemRepository.save(...) がそのまま使える
